# Version Control（版本控制）

## Git Workflow

- 使用有意義的 commit messages
- 遵循專案的 branching strategy
- Pull Request 前確保通過所有測試

## Commit Messages

- 格式：`type(scope): 繁體中文描述`
- Types: `feat`, `fix`, `refactor`, `test`, `docs`, `chore`
- 範例：
  - `feat(user): 新增註冊端點`
  - `fix(order): 修正重複建立問題`
  - `refactor(payment): 抽取策略模式`
  - `test(product): 補齊庫存扣減測試`
