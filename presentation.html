Part 02 — MCP

MCP 是什麼？

讓 AI 能操作外部工具的標準介面。

把 AI 想成很聰明的新人，MCP 就是給他各種系統的帳號和權限。沒有 MCP，他只能告訴你怎麼做；有了 MCP，他能直接進去做。

互動流程

👤
You — 下指令
「幫我查 GitHub issue」
⚡
Claude Code (MCP Client)
整合工具清單送出
AI
LLM 思考決策
決定呼叫哪個工具 + 參數
⚡
Claude Code
去敲 MCP Server 的門
🔧
MCP Server (GitHub)
執行 → 回傳結果
👤
You — 看到答案
此循環可跑多次直到完成

Part 03 — Agent Skills

Agent Skills
是什麼？

預先寫好的工作指令包，AI 在對的時機自動呼叫，你也可以用 /指令 手動觸發。

Skills 就像公司的 SOP——與其每次都跟新人說一遍流程，不如把 SOP 寫好，叫他照做就好。

怎麼觸發

AI 自動

你說「我想討論報告內容」→ AI 判斷這是設計討論 → 自己呼叫 /brainstorming

手動呼叫

直接輸入 /test-driven-development → AI 照 TDD 流程走，不需要任何額外說明。

注意事項

AI 工具的資安邊界

數據訓練疑慮

呼叫商業 API 時，輸入的內容可能被用於模型訓練，這幾乎無法完全避免。

防範原則

  • 絕對不輸入：個資、密碼、API keys、商業機密
  • 程式碼中的敏感邏輯先抽換成假資料再貼入
  • 企業敏感專案考慮使用私有部署或企業版合約（明確不訓練）
系統權限控管

Agent 能力越強，誤操作的代價越高。最小權限原則在 AI 工具上同樣適用。

防範原則

  • 每次 Agent 要執行指令時，看清楚再確認，不要全部 auto-approve
  • 用 Sandbox / 隔離環境跑不確定的 Agent 任務
  • 生產環境的寫入、刪除操作，永遠要人工二次確認
  • MCP 只裝需要的，裝越多 = 暴露面越大
