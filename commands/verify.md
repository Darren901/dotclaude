執行完整的專案驗證流程，確認所有品質關卡都通過後才宣告成功。

## 執行步驟

依序執行以下指令，任一步驟失敗則立即停止並回報錯誤：

1. **編譯驗證**
   ```bash
   mvn clean compile
   ```

2. **執行所有測試**
   ```bash
   mvn test
   ```

3. **產生覆蓋率報告並驗證門檻**
   ```bash
   mvn jacoco:report
   ```

## 回報格式

完成後用繁體中文輸出結構化報告：

```
## Verify 結果

### 編譯
✅ 成功 / ❌ 失敗（附錯誤訊息）

### 測試
✅ X 個通過 / ❌ X 個失敗 / ⚠️ X 個跳過
（列出失敗的測試名稱與原因）

### 覆蓋率
- Line Coverage：XX%（門檻：80%）✅ / ❌
- Branch Coverage：XX%（門檻：80%）✅ / ❌

### 結論
🟢 PASS - 可以進行 Code Review / Commit
🔴 FAIL - 需修正以下問題：...
```

## 重要原則

- 不得跳過任何步驟
- 不得在指令實際執行前宣告成功
- 覆蓋率若低於 80% 視為失敗，需補測試
- 若非 Maven 專案，自行偵測對應的建構工具（npm、gradle 等）並調整指令
